# 🎯 坦克建模 & 场地建模优化方案

## 当前问题诊断

经过深入代码审查，发现以下核心问题：

### 坦克建模现状

| 问题 | 严重程度 | 详情 |
|------|----------|------|
| **全部使用 Box/Cylinder 基元拼装** | 🔴 严重 | [TankVisualFactory.ts](file:///c:/work/github/tank-battle/src/game/graphics/TankVisualFactory.ts) 中仅用 `BoxGeometry` + `CylinderGeometry` 堆叠，没有任何曲面、倒角、细节 |
| **只有 4 款坦克有专属造型** | 🟠 中等 | Tiger I / Panther / T-34/85 / Sherman 有手写模型，其余 10 种坦克走 `buildFallback` —— 只是个方盒子 |
| **无履带轮组细节** | 🔴 严重 | 履带只是两条拉长的 Box，没有负重轮、驱动轮、惰轮 |
| **无任何装饰细节** | 🟠 中等 | 没有舱门、天线、工具箱、排气管、观瞄镜等辅件 |
| **GLTF 模型管线已搭好但无模型文件** | 🟢 利好 | `ModelLoader.ts` 已完成 GLTF 加载/分段/缩放逻辑，`public/models/tanks/` 目录为空 |

### 场地建模现状

| 问题 | 严重程度 | 详情 |
|------|----------|------|
| **地面几乎是平面** | 🔴 严重 | `sampleGroundHeight()` 只有微弱 sin/cos 起伏（最大 ±0.5m），物理碰撞面是无穷大平面 `CANNON.Plane` |
| **建筑全是 Box + ConeGeometry** | 🟠 中等 | 厂房/仓库/废墟 = 方盒子 + 屋顶锥体，无门窗、无破损纹理 |
| **植被仅有锥体树** | 🟡 轻微 | 180 个 `ConeGeometry(0.16, 1.2, 6)` instanced mesh，看起来像棵铅笔 |
| **地面纹理程序生成** | 🟡 轻微 | 512px Canvas 随机圆点画，分辨率和真实感都不够 |
| **3 张战场地图布局固定** | 🟡 轻微 | 可玩但缺少变化 |

---

## 三个优化方案

---

### Option A：纯代码程序化精修（Procedural Enhancement）

> **核心思路**：不引入外部资产，100% TypeScript 代码优化，用更高级的程序化几何和材质提升效果

#### 🔧 坦克改进

1. **精细化车体轮廓**
   - 使用 `ExtrudeGeometry` + 自定义 `Shape` 画出每种坦克的实际侧面/俯视轮廓（圆弧前装甲、斜面侧裙等）
   - 车体不再是 Box，而是 8-12 个面的多边形挤压体
   - 炮塔使用 `LatheGeometry` 生成更圆润的铸造炮塔（T-34、Sherman）或棱角分明的焊接炮塔（Tiger I）

2. **添加负重轮组**
   - 每侧 5-8 个 `CylinderGeometry`（r=0.32 ~ 0.38）均匀分布
   - 驱动轮（前/后）、惰轮（后/前）直径略大
   - 履带改用 `PlaneGeometry` wrap 贴合轮组底部弧线

3. **添加装饰细件**
   - 舱门（小圆柱 + 凸起）、天线（细 Cylinder）、排气管、手柄、工具箱（小 Box）
   - 每种坦克定义一个 `DetailParts[]` 阵列

4. **改进迷彩纹理**
   - 提升到 2048px
   - 按坦克类型/阵营区分更丰富的迷彩图案

#### 🏞️ 场地改进

1. **增强地形起伏**
   - 引入 Fractal Brownian Motion (FBM) 噪声取代简单 sin/cos
   - 添加弹坑（随机扣洞）和土堤
   - 物理碰撞面改用 `CANNON.Heightfield` 与视觉高度匹配

2. **升级建筑几何**
   - 为墙体添加窗户（挖洞 Boolean）或用 `ShapeGeometry` 画窗孔
   - 屋顶使用 `BufferGeometry` 构建双坡屋顶
   - 加入门框、梁柱等结构细节

3. **升级植被**
   - 树冠改用多层 `DodecahedronGeometry`（低多边形风格）叠加
   - 添加灌木丛 instanced mesh（`SphereGeometry` 压扁）
   - 添加草叶 instance billboard

4. **地面纹理提质**
   - 提升到 1024px + 多 octave 程序化噪声
   - 不同区域（草/泥/碎石/路面）各自生成专用纹理

#### 📊 评估

| 指标 | 评分 |
|------|------|
| 坦克友好度（可爱/辨识度） | ⭐⭐⭐ |
| 场地真实感 | ⭐⭐⭐ |
| 开发工期 | **3-5 天** |
| 性能影响 | 低（几何增加可控） |
| 维护成本 | 中（代码量大幅增加） |
| 是否需要美术资源 | ❌ 不需要 |

---

### Option B：引入免费 GLTF 模型 + 高质纹理贴图（Asset-Driven）

> **核心思路**：利用已搭建好的 `ModelLoader` 管线，加载社区免费 GLTF 坦克模型 + 真实贴图

#### 🔧 坦克改进

1. **获取免费坦克模型**
   - 来源：[Sketchfab CC BY](https://sketchfab.com) / [OpenGameArt](https://opengameart.org) / [poly.pizza](https://poly.pizza)
   - 目标：为 14 种坦克各获取一个 GLTF/GLB 模型
   - 要求：每个模型分段标注 `Hull` / `Turret` / `Gun` 节点
   - 文件放入 `public/models/tanks/<tank-id>.glb`

2. **完善 ModelLoader 管线**
   - 为全部 14 种坦克补全 `MODEL_CONFIGS` 配置
   - 增强节点识别逻辑（支持更多命名约定）
   - 添加 LOD 支持：远距离用当前 Fallback Box 几何，近距离切换 GLTF

3. **增强材质系统**
   - `CamouflageGenerator` 生成的迷彩贴图仍然可覆盖到 GLTF 模型上
   - 添加模型自带贴图与程序化迷彩的混合模式

4. **可选：卡通化/友好化后处理**
   - 加入 Outline post-processing（描边），让坦克更卡通友好
   - 略微放大炮塔/缩短车体比例，营造 Q 版感

#### 🏞️ 场地改进

1. **替换建筑为 GLTF 模型**
   - 获取免费低多边形建筑资产包（rural house, factory, ruin）
   - 替换 `createStructureShell()` 中的 Box 构建逻辑

2. **引入地面贴图集**
   - 从 [ambientCG](https://ambientcg.com) 获取免费 PBR 贴图（草地、泥地、碎石、沥青）
   - 替换程序化 Canvas 纹理

3. **增强植被**
   - 引入低多边形树模型（3-5 种变体）
   - 添加草地 billboard 对象

#### 📊 评估

| 指标 | 评分 |
|------|------|
| 坦克友好度（可爱/辨识度） | ⭐⭐⭐⭐⭐ |
| 场地真实感 | ⭐⭐⭐⭐⭐ |
| 开发工期 | **5-8 天**（含模型寻源 & 适配） |
| 性能影响 | 中（首次加载增大，需 LOD 优化） |
| 维护成本 | 低（模型替换方便） |
| 是否需要美术资源 | ✅ 需要（免费资源即可） |

> [!IMPORTANT]
> 此方案达到最佳视觉效果，但需要花时间搜索、下载并适配每个模型。模型的 license 需确认为 CC0 / CC BY。

---

### Option C：混合方案 — 程序化 Low-Poly 风格化 + 选择性 GLTF（推荐）

> **核心思路**：采用统一的 Low-Poly 风格化美术方向，坦克用精细程序化几何+风格化着色器，场地选择性引入少量关键 GLTF 资产

#### 🔧 坦克改进（程序化风格化）

1. **设计统一 Low-Poly 风格语言**
   - 坦克整体保持棱角分明、但比现在的 Box 精细 3-4 倍
   - 车体用 8-16 面的 `BufferGeometry` 手动构建，带倒角、斜面
   - 炮塔用 `LatheGeometry` + 6-8 段，略带圆润感
   - **友好度提升**：略微 Q 版化比例（炮塔大 15%、车体短 10%、负重轮略大）
   
2. **为核心坦克手写精细模型**
   - 优先覆盖 4 大阵营各 1 款代表坦克（Tiger I, Sherman, T-34/85, Cromwell）
   - 其余坦克基于同阵营代表款的变体（调整比例/细节）

3. **添加风格化着色器**
   - 自定义 `ShaderMaterial`：Cel-Shading（卡通着色）+ 描边
   - 保持 PBR 迷彩底色但叠加 2-3 阶色调映射
   - 效果类似《坦克世界闪电战》卡通渲染模式

4. **为全部 14 种坦克添加负重轮组 + 装饰小件**
   - 共享轮组模板（3 种类型：大轮/小轮/交错）
   - 每种坦克定义 `details: { antenna, toolbox, exhaustPipe, hatch }` 开关

#### 🏞️ 场地改进（混合方式）

1. **地形引入 Heightfield**
   - FBM 噪声生成高度图
   - 物理面同步为 `CANNON.Heightfield`
   - 添加弹坑和沟壑的程序化凹坑

2. **建筑用程序化 + 3-5 个关键 GLTF 资产**
   - 程序化升级：墙体加窗户/门洞、屋顶加烟囱/天线
   - GLTF 补充：1 个教堂/钟楼、1 个坍塌建筑、1 个谷仓（免费 low-poly 模型）
   
3. **植被全面升级**
   - 树木：多层 `DodecahedronGeometry` 树冠 + `CylinderGeometry` 树干（统一风格）
   - 灌木：扁 `SphereGeometry` instanced
   - 草地：billboard instanced mesh（2 种变体）

4. **地面纹理改进**
   - 1024px PBR 程序化贴图（保留自生成优势）
   - 增加地面碎片 detail（小石头、弹壳 instance）

#### 📊 评估

| 指标 | 评分 |
|------|------|
| 坦克友好度（可爱/辨识度） | ⭐⭐⭐⭐ |
| 场地真实感 | ⭐⭐⭐⭐ |
| 开发工期 | **4-6 天** |
| 性能影响 | 低-中（风格化可控） |
| 维护成本 | 低（风格统一好维护） |
| 是否需要美术资源 | 少量（3-5 个免费 GLTF） |
| 视觉风格一致性 | ⭐⭐⭐⭐⭐ |

> [!TIP]
> 此方案的**最大优势**是风格统一。纯程序化部分和少量 GLTF 资产统一在 Low-Poly 风格化语言下，不会出现"有的坦克写实、有的坦克方盒子"的割裂感。卡通着色器让坦克看起来"友好可爱"而非冰冷硬核。

## 三方案对比总览

| 维度 | Option A（纯程序化） | Option B（GLTF 驱动） | Option C（混合风格化）✨ |
|------|---------------------|----------------------|------------------------|
| 坦克可爱度 | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ |
| 场地真实感 | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ |
| 工期 | 3-5 天 | 5-8 天 | 4-6 天 |
| 外部依赖 | 无 | 14+ 模型文件 | 3-5 模型文件 |
| 性能 | 最优 | 需 LOD 优化 | 优 |
| 风格一致性 | 好 | 可能不一致 | 最好 |
| 适合场景 | 想快速迭代 | 追求最高视觉上限 | 平衡美感与效率 |

## 个人推荐

> [!IMPORTANT]
> **推荐 Option C（混合风格化）**
> 
> 理由：
> 1. 你的目标是"友好的坦克形象" —— 卡通风格化着色器 + 微调 Q 版比例最能达到这个效果
> 2. 你的目标是"真实的场地" —— FBM 地形 + 少量精选 GLTF 建筑即可大幅提升
> 3. 风格统一是最重要的设计原则，混合方案在这点最好
> 4. 工期适中、性能友好，且保留了未来替换为更精细模型的通道
